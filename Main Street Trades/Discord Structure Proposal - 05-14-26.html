Restructure the server so members can scroll, browse, and pick their guy.

Today the server mixes machine feeds (UOV scanner, insider buys) with human analysts (Charlie). It also buries paid tools in one category and analysts in another. The fix: two clear top-level sections. Paid Tools / Signals = the data we monitor. Partner Analysts = the humans giving picks, each with their own visual header. As you bring on more partners, you just drop another divider.

🛠️ Paid Tools / Signals = everything automated (the data feeds you pay vendors for). 🤝 Partner Analysts = humans. Each analyst gets a locked, view-only divider channel above their feeds — easy to scroll, easy to extend.

Why this structure works

01

Two mental buckets, not five

New members understand "tools" vs. "people" instantly. Calendar, Short Term, and Signals all collapse into one bucket because they all answer the same question — what is the machine watching?

02

Each analyst is a brand

The divider channel ("━━ charlie-plattus ┃ zip-trader ━━") sits inside the sidebar as a visible header. It tells the member: this isn't a feed — this is a person with their own service. You'd never confuse Charlie with the UOV scanner again.

03

Drop-in slot for the next partner

When you bring on analyst #2 (or #5), you don't restructure anything. You add a new divider + their feed channels under 🤝・Partner Analysts. Members already know to scroll there to pick their guy.

The "header divider" pattern

Discord doesn't support nested categories. The hack everyone uses: a locked, view-only text channel whose name is just a visual separator. Members can't post in it, can't even hear notifications. It just acts as a section header inside a category.

1
Naming — use unicode block characters so the channel name reads as a header, not a normal channel:
━━charlie-plattus┃zip-trader━━
━━jane-doe┃macro-investor━━
2
Permissions — @everyone is denied SEND_MESSAGES and ADD_REACTIONS. Only Founder + Mod can post. It's purely cosmetic.
3
Per-analyst real estate — each analyst gets a divider plus 1–3 feed channels underneath (ideas / watchlist / Q&A). Charlie's gets #charlie-options-ideas today. Easy to add #charlie-watchlist or #ask-charlie later without restructuring.
4
Discovery — pin a single message in #start-here linking to each analyst's divider with a one-line pitch ("Charlie Plattus — options swing trader, Zip Trader founder, posts 1–3 ideas/day"). Members tap the link, see the divider, decide if they want to follow.

What changes vs. what stays

Changes

  • Delete categories: 📅・CALENDAR, ⚡・SHORT TERM, 📡・SIGNALS.
  • Create category: 🛠️・PAID TOOLS / SIGNALS. Move all 6 feed channels into it.
  • Create category: 🤝・PARTNER ANALYSTS.
  • Create divider channel: ━━charlie-plattus┃zip-trader━━ (locked).
  • Move #charlie-options-ideas under the Charlie divider.
  • Update start-here pin to explain the new sections.

Stays the same

  • All existing channels keep their permissions, history, webhooks, and member subscriptions.
  • Roles (Founder / Mod / VIP / Premium / Free) and their access rules don't change.
  • Charlie's mirror webhook keeps working — same channel ID.
  • Discussion, Live Rooms, Perks, VIP Only categories untouched.

Implementation: one idempotent node script (~150 LOC). Same pattern as the existing setup-server.js + add-emojis.js — re-runnable, no data loss, can be reverted by dragging in the Discord UI. Say the word and I'll wire it up.
